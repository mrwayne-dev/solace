📝 Week 1 Rundown: JavaScript Fundamentals and Logic

1. The Core Building Blocks 🧱

This section focused on the very basic syntax required to write working code.

    Syntax and Setup: We established how to link JavaScript to an HTML file using the <script> tag and how to use the browser console for debugging (console.log()).

    Variables:

        let vs. const: Learned the difference between the two primary ways to declare variables: const for values that won't change (constants) and let for values that will be reassigned.

        Template Literals: Mastered the use of backticks ` to embed variables directly into strings using ${variable}.

    Functions: Defined and executed basic functions to encapsulate a block of code, allowing it to be reused or called upon by an event.

2. Data Flow and Calculations 🔢

This was a major milestone, as the student transitioned from static, hardcoded data to interactive programs.

    User Input (Inputs over Prompts): Moved beyond the simple prompt() to capture data from structured HTML Input Fields (<input id="...">).

        Key Skill: Accessing the user's input value using .value (e.g., document.getElementById("myInput").value).

    Crucial Type Conversion: Identified and solved the fundamental problem that input values are always strings.

        Tool: Used the Number() function to explicitly convert string input into numerical data before performing any math, ensuring 5 + 5 equals 10 instead of '55'.

    The Math Object: Learned to perform complex calculations using the built-in Math object.

        Methods: Used Math.pow() (for exponents), Math.round() (for cleaning up decimals), and Math.random() (for generating random numbers, like a digital dice roll).

3. Web Interactivity (The DOM) 💻

The student learned how to make their JavaScript code directly affect what the user sees in the browser.

    DOM Manipulation: Used document.getElementById() to select elements on the page.

        Displaying Data: Changed the visible content of an HTML element using the .innerHTML property.

    Event Handling: Connected JavaScript functions to HTML events using the onclick attribute on buttons.

    Execution Order (Scope): Understood the difference between code that runs immediately on page load (outside the function) and code that runs only when called (inside the function), which was crucial for debugging the multiplier project.

4. Decision Making (Conditionals) 🧠

The final and most important concept of the week—giving the program intelligence.

    Comparison Operators: Learned the operators used to ask true/false questions:

        >, <, >=, <=.

        === (Strict equality, comparing both value and type).

    The if/else Structure: Built programs that can choose between two or more paths of execution.

        Used the if, else if, and else statements to check different conditions sequentially (e.g., the Pass/Fail checker or the Ticket Price App).

    Logical Operators: Combined multiple conditions for complex rules:

        && (AND): Requires both conditions to be true.

        || (OR): Requires at least one condition to be true.
