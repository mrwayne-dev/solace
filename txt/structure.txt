The wallet.php file serves as the central controller for all wallet-related operations within the HealthRunCare ecosystem. It shows balances, manages deposits, withdrawals, and transaction tracking for authenticated users through a single, secure API endpoint. When a user initiates a deposit from the dashboard by inputing the amount and clicking the deposit now button (an email(Deposit Request Initiated) is sent to the user and our toast message(Redirecting) or loader shows), the frontend sends a request to this backend with the deposit amount and selected method. If the method is Secure Exchange, the request is routed through the create_crypto_payment.php file under /api/payments, which connects to NOWPayments to generate a cryptocurrency payment link and returns it to the frontend for redirection. Once the user completes the payment, NOWPayments’ webhook verifies (nowpayment_webhook.php) it and automatically credits the user’s wallet and the user receives an email saying Deposit Request Paid For manual deposit methods like wire transfer or cash mailing, Once the user puts in an amount and selects either method and clicks the Deposit Now button, a toast message is shown saying "Support will provide details shortly", and an email(Deposit Request Initiatied) is sent to the user and the system creates a pending transaction in the transactions table and sends an email to admin — notifying the admin to provide payment details.
